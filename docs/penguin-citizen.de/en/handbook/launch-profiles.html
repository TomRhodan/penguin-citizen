Handbook

    Launch Profiles

    Save and switch between named launch configurations. Each profile bundles a Wine runner and every option from the Launch page so swapping between "Stable" and "Wayland Test" is one click instead of a manual reset.

    What is a Launch Profile?

    A Launch Profile is a saved snapshot of:

    • The chosen Wine runner
    • All performance & graphics options from the Launch page (ESync, FSync, DXVK Async, GPU-specific flags, Gamescope, MangoHUD, custom env vars, …)

    One profile is always active. The active profile drives the actual launch when you press Play, both from the Launch page and from the Dashboard.

    The Profile Wizard

    The fastest path from a fresh install to a working game. Open it from the Dashboard:

    • The ⚙ Set up & Launch button (when no usable profile exists yet)
    • The small ⚙ Profile Wizard button below the play button (in the Ready state)

    The wizard is a single-step modal:

    1. Pick a Wine runner from your installed set. The wizard automatically pre-selects a recommended runner: a LUG-Helper build first (most reliable for SC), then any GE-Proton build, then the alphabetically first installed runner.
    2. Pick a profile name (defaults to My Setup; conflicts with existing names show a red border with inline validation).
    3. Choose how to finish:
      • Save & Done — closes the wizard and returns to the Dashboard. The new profile is active and ready to launch.
      • Customize — saves the profile and jumps to the Launch page so you can fine-tune individual toggles before playing.
    Wizard profiles default to wayland=false on purpose — Wayland for SC is still rough at the edges and the wizard sells itself as the safe path. Power users can toggle Wayland on later via the Launch page.

    Profile Management Page

    Open Launch Profiles from the sidebar to access the management UI: a card grid of all your profiles, plus the Wine Runner Fallback section.

    Profile Cards

    Each card shows the profile name (with a star ★ on the active one), an optional description, the runner, and creation/update timestamps. The available actions per card:

    • Set Active — only on non-active profiles. Switching while the current profile has unsaved changes triggers a confirm dialog so you don't lose your tweaks.
    • Rename — opens a styled prompt. Names must be unique across all profiles; collisions are rejected by the backend.
    • Edit description — optional free-text note that shows up in the card header. Leave empty to clear.
    • Duplicate — deep copy of the profile body under a new name. Useful for "Stable + DXVK HUD on top" without touching the original.
    • Delete — disabled for the active profile (switch first) and for the very last remaining profile (the system always keeps at least one).

    Create New Profile

    The + Create new profile from current settings button at the top snapshots the current Launch page state into a new profile. Different from the wizard, which always uses safe defaults.

    Wine Runner Fallback

    The fallback is a global runner that takes over at launch time if the active profile's runner has been uninstalled. Configure it via the dropdown at the bottom of the Launch Profiles page.

    • The dropdown lists all currently installed runners. The selected one is validated when saved — if it disappears later, the launch flow surfaces an error toast.
    • If the active profile's runner is missing AND a fallback is configured, launching emits a runner-fallback-used event and a toast indicates the fallback was used.
    • Set the dropdown to — none — to disable the fallback. Without it, a missing profile runner aborts the launch with a clear error pointing at this page.

    Dirty Workflow on the Launch Page

    When you toggle any option on the Launch page, the working state diverges from the saved profile. The profile bar reflects this:

    • Pill changes from green Saved to amber Unsaved Changes
    • Update Profile button — commits the working state into the active profile
    • Revert button — restores the saved profile values, discarding the tweaks

    Launching with unsaved changes works fine — the launch uses the live working state. The profile just stays dirty until you commit or revert.

    Wine Runner Page Cross-Reference

    The Wine Runners page shows per-runner usage badges so you can see at a glance which profiles depend on a given runner before deleting it: Used by N profile(s) and Fallback badges. Hover the "Used by" badge to see the profile names.
