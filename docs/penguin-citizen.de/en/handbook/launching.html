Handbook

    Launching the Game

    Configure GPU-specific options, enable Gamescope and GameMode, and start Star Citizen with a single click.

    Launch Profile Bar

    Above the launch bar, the profile bar exposes the active Launch Profile — a saved bundle of Wine runner + every option on this page. From left to right it shows:

    • Profile dropdown with all saved profiles. The active one is starred (★). Switching takes effect immediately.
    • Dirty / Saved pill — green Saved when the working state matches the active profile, amber Unsaved Changes when any toggle on this page differs.
    • Wine runner selector — lists installed runners; lets you change the runner for the active profile without leaving the page. Profiles whose runner is no longer installed get a warning pill.
    • Update Profile / Revert — only visible when dirty. Update writes the current state back into the active profile; Revert restores the saved values and discards your tweaks.
    • Manage Profiles link — opens the dedicated Launch Profiles page for create / rename / duplicate / delete actions and the global Wine Runner Fallback.
    Switching profiles while dirty surfaces a confirm dialog so you don't lose tweaks accidentally. Pick Discard & Switch to drop them, or Cancel to stay on the current profile.

    Launch Bar

    Below the profile bar, the compact launch bar shows the Play/Stop button, the runner currently in use (taken from the active profile), the Wine prefix path, and a color-coded status indicator (ready, running, error). While the game is running, the button switches to a Stop button.

    GPU Detection

    Penguin Citizen automatically detects your graphics card (NVIDIA, AMD, or Intel) when loading the launch page. GPU-specific option cards are only enabled for the detected GPU - non-matching cards are greyed out with a notice.

    Option Cards

    All launch options are organized in a collapsible 3-column card grid. Each card shows badges with active options when collapsed. Click a card to expand it and adjust settings.

    Performance

    • ESync - Eventfd-based synchronization for Wine. Reduces CPU overhead. Recommended.
    • FSync - Futex-based synchronization (requires kernel 5.16+). More efficient than ESync. Enable both if possible.
    • DXVK Async - Asynchronous shader compilation. Reduces stuttering during gameplay. Highly recommended.
    • GameMode - Launches the game with gamemoderun for optimized CPU/GPU performance. Requires the gamemode package on your system. If not installed, the option is disabled with a warning.

    Display

    • HDR - High Dynamic Range output. Requires an HDR-capable monitor and compositor support (e.g., KDE Plasma 6 with HDR enabled).
    • FSR - AMD's FidelityFX Super Resolution. Renders at lower resolution and upscales for better performance.

    Overlays

    • MangoHUD - Performance overlay showing FPS, CPU/GPU usage, temperatures, and frame times.
    • DXVK HUD - DXVK debug overlay for rendering diagnostics.

    NVIDIA

    Only active when an NVIDIA GPU is detected.

    • DLSS 4.0 - NVIDIA's AI-powered upscaling and frame generation for higher frame rates.
    • Smooth Motion - Frame interpolation for smoother visuals.
    • G-Sync - Synchronizes the monitor refresh rate with GPU output to eliminate tearing.

    AMD

    Only active when an AMD GPU is detected.

    • RADV Zero VRAM - Sets radv_zero_vram to fix rendering artifacts on RADV drivers.
    • Nogttspill - Disables GTT spillover. Can improve stability on AMD GPUs with limited VRAM.

    Troubleshooting

    • In-Process GPU - Sets --in-process-gpu to fix black or white launcher windows.
    • Vulkan Mailbox - Enables the Vulkan mailbox present mode for lower latency.
    • HDR WSI Layer - Enables the HDR WSI layer for extended HDR support via the Vulkan interface.

    Wayland

    • Enable Wayland - Toggle to use Wayland instead of XWayland. Automatically disabled if fractional scaling is detected.
    • Primary Monitor - Select which monitor to use for the game. Connected monitors are auto-detected with their resolutions. A refresh button allows re-detecting monitors.
    Wayland support is experimental. If you experience issues, disable it to fall back to XWayland. Fractional scaling (e.g., 125%, 150%) is not compatible with native Wayland mode.

    Gamescope

    Gamescope is a standalone Wayland compositor by Valve that provides a controlled rendering environment for games. It is commonly used for HDR support, resolution management, and better fullscreen control.

    • Enable Gamescope - Launches the game inside Gamescope. Requires gamescope to be installed on your system. If not installed, the card displays an installation warning.
    • Resolution - Width and height of the Gamescope output (default: 2560x1440).
    • HDR - Enables HDR output within Gamescope.
    • Force Grab Cursor - Forces mouse cursor capture inside the game window.
    • Keyboard Grab - Captures keyboard input so system shortcuts don't interfere with the game.
    When collapsed, the Gamescope card shows badges for the current status: "Active", "HDR", the configured resolution, or "Disabled" / "Not Installed".

    GPU & CPU

    • GPU Device Filter - Select a specific Vulkan device for launching. Useful for hybrid systems (e.g., laptop with integrated and dedicated GPU). Available devices are auto-detected.
    • CPU Topology - Sets the Wine CPU topology (e.g., core and thread mapping) for optimized game performance.

    Custom Environment Variables

    For advanced users, you can define custom environment variables that will be set when launching Star Citizen. Each variable has:

    • An enable/disable toggle to quickly activate or deactivate without deleting
    • A KEY field (automatically converted to uppercase, only A-Z, 0-9, and underscore allowed)
    • A VALUE field

    A conflict warning badge appears if a custom variable would override one of the built-in options (like ESync or FSync).

    Log Output

    The log panel at the bottom shows real-time output from the game launch process. It streams log lines as they arrive and auto-scrolls to the latest entry. This is helpful for diagnosing launch failures or monitoring the startup sequence.

    All launch options are disabled while the game is running to prevent configuration changes during gameplay. They become editable again once the game exits.
