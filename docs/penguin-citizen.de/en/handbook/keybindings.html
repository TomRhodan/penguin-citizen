Handbook

    Keybindings

    View and edit controller bindings, manage devices, and fine-tune joystick axes.

    Binding Matrix

    The keybindings page displays all Star Citizen actions in a table. Each row is an action; columns represent your connected devices — keyboard, mouse, and any joysticks or gamepads. Action categories are listed in the sidebar on the left.

    Filtering and Search

    • Category Sidebar — Click a category (e.g. Flight Movement, Targeting) to jump to it or collapse it.
    • Search — Type in the search box to filter actions by name in real-time.
    • Device Tabs — Show only the columns for a specific device type (Keyboard, Mouse, Joystick, Gamepad), or show all at once.
    • Bound Only — Toggle to hide all actions that have no bindings yet, so you only see what is actually mapped.

    Editing a Binding

    Each cell in the matrix belongs to one action and one device. To change a binding:

    1. Click the + button in an empty cell to add a new binding for that action and device.
    2. Click the label of an existing binding pill to edit it.
    3. The appropriate editor dialog opens (see below).
    4. Click Save to apply, Reset to Default to restore the original, or Cancel to discard.

    To remove a binding entirely, click the × button on the binding pill.

    Changes are written directly to your active Star Citizen profile. Make sure a profile is loaded before editing bindings — if no profile is active, the save button will show an error.

    Keyboard & Joystick Bindings

    For keyboard and joystick/gamepad bindings, clicking a cell opens the live capture dialog. Simply press the key, button, or move the axis you want to assign — Penguin Citizen detects the input automatically. If the input is already used by another action, a conflict warning is shown.

    Mouse Bindings

    Mouse bindings use a dedicated static selection dialog instead of live capture. Live capture is not possible for the mouse because the mouse cursor is needed to operate the application itself.

    The dialog shows all 11 Star Citizen-compatible mouse inputs grouped in three columns:

    • Movement Axes — X-Axis (horizontal) and Y-Axis (vertical)
    • Scroll Wheels — Scroll up/down and horizontal scroll wheel right/left
    • Buttons — Left click, Right click, Middle click, Side button back, Side button forward

    Click any row to select it — the Current badge at the top updates immediately. Click Save to apply.

    Joystick Management

    Penguin Citizen detects connected joysticks and displays them in the device sidebar. You can drag and drop joysticks to reorder their instance numbers, which controls how Star Citizen assigns js1_, js2_, etc. to your devices.

    Joystick instance numbers in Linux can differ from what Star Citizen expects. If your bindings appear to be on the wrong device, try reordering your joysticks here.

    Axis Tuning

    Joystick axis bindings show a small tuning button (∧) next to the binding pill. Click it to open the axis tuning editor for that specific action:

    • Deadzone — Ignore small movements around the centre position. Useful for joysticks with drift.
    • Sensitivity Curve — Choose a response curve (Linear, Exponential, etc.) that controls how physical movement translates to in-game movement.
    • Axis Inversion — Flip the axis direction.
    • Sensitivity Multiplier — Scale the overall sensitivity up or down.

    Tuning settings are stored per joystick device, so swapping USB ports does not reset them.

    Mouse axis bindings (X-Axis, Y-Axis) do not have a tuning button — Star Citizen handles mouse sensitivity through its own in-game settings rather than the profile XML.
