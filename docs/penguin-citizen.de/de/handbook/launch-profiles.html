    Launch-Profile

    Speichere und wechsle zwischen benannten Start-Konfigurationen. Jedes Profil buendelt einen Wine-Runner und alle Optionen der Startseite — Umschalten zwischen "Stable" und "Wayland Test" ist ein Klick statt manueller Reset.

    Was ist ein Launch-Profil?

    Ein Launch-Profil ist ein gespeicherter Snapshot aus:

    • dem gewaehlten Wine-Runner
    • allen Performance- & Grafikoptionen der Startseite (ESync, FSync, DXVK Async, GPU-spezifische Flags, Gamescope, MangoHUD, eigene Umgebungsvariablen, …)

    Genau ein Profil ist immer aktiv. Das aktive Profil bestimmt den tatsaechlichen Start, sowohl von der Startseite als auch vom Dashboard.

    Der Profil-Assistent

    Der schnellste Weg von der frischen Installation zum laufenden Spiel. Vom Dashboard aus zu oeffnen:

    • Den ⚙ Einrichten & Starten-Button (wenn noch kein nutzbares Profil existiert)
    • Den kleinen ⚙ Profil-Assistent-Button unter dem Play-Button (im Bereit-Zustand)

    Der Assistent ist ein Single-Step-Modal:

    1. Waehle einen Wine-Runner aus deinem installierten Set. Der Assistent waehlt automatisch einen empfohlenen vor: zuerst einen LUG-Helper-Build (am verlaesslichsten fuer SC), dann einen GE-Proton-Build, sonst den alphabetisch ersten installierten Runner.
    2. Vergib einen Profilnamen (Standardwert My Setup; Konflikte mit existierenden Namen markieren das Feld rot mit Inline-Validation).
    3. Waehle den Abschluss:
      • Save & Done — schliesst den Assistenten und kehrt zum Dashboard zurueck. Das neue Profil ist aktiv und startbereit.
      • Customize — speichert das Profil und springt zur Startseite, damit du einzelne Toggles vor dem Spielen feintunen kannst.
    Assistent-Profile haben absichtlich wayland=false als Standard — Wayland fuer SC ist noch unausgereift, und der Assistent verkauft sich als sicherer Pfad. Power-User koennen Wayland spaeter ueber die Startseite einschalten.

    Profil-Verwaltungsseite

    Oeffne Launch-Profile aus der Sidebar fuer das Verwaltungs-UI: ein Card-Grid aller Profile plus den Wine-Runner-Fallback.

    Profil-Karten

    Jede Karte zeigt den Profilnamen (mit Stern ★ beim aktiven), eine optionale Beschreibung, den Runner sowie Erstellungs- und Aktualisierungs-Zeitstempel. Verfuegbare Aktionen pro Karte:

    • Set Active — nur bei nicht-aktiven Profilen. Beim Wechsel mit ungespeicherten Aenderungen erscheint ein Bestaetigungsdialog, damit nichts verloren geht.
    • Rename — oeffnet einen styled Prompt. Namen muessen ueber alle Profile eindeutig sein; Kollisionen werden vom Backend abgelehnt.
    • Edit description — optionale Freitext-Notiz, die im Card-Header angezeigt wird. Leer lassen zum Loeschen.
    • Duplicate — Tiefenkopie des Profil-Bodys unter neuem Namen. Nuetzlich fuer "Stable plus DXVK HUD obendrauf", ohne das Original anzufassen.
    • Delete — deaktiviert beim aktiven Profil (vorher umschalten) und beim allerletzten Profil (das System haelt immer mindestens eines).

    Neues Profil anlegen

    Der + Create new profile from current settings-Button oben snapshottet den aktuellen Stand der Startseite in ein neues Profil. Anders als der Assistent, der immer mit sicheren Voreinstellungen arbeitet.

    Wine-Runner-Fallback

    Der Fallback ist ein globaler Runner, der zur Startzeit einspringt, wenn der Runner des aktiven Profils deinstalliert wurde. Konfiguriere ihn ueber das Dropdown am unteren Rand der Launch-Profile-Seite.

    • Das Dropdown listet alle aktuell installierten Runner. Der ausgewaehlte wird beim Speichern validiert — verschwindet er spaeter, surfaced der Start-Flow eine Fehler-Toast.
    • Wenn der Runner des aktiven Profils fehlt UND ein Fallback konfiguriert ist, sendet der Start ein runner-fallback-used-Event und ein Toast meldet die Verwendung des Fallbacks.
    • Setze das Dropdown auf — none —, um den Fallback zu deaktivieren. Ohne Fallback bricht ein fehlender Profil-Runner den Start mit einem klaren Fehler ab, der auf diese Seite verweist.

    Dirty-Workflow auf der Startseite

    Sobald du eine Option auf der Startseite umlegst, weicht der Arbeitszustand vom gespeicherten Profil ab. Die Profil-Leiste reflektiert das:

    • Pille wechselt von gruen Saved zu orange Unsaved Changes
    • Update Profile-Button — schreibt den Arbeitszustand zurueck ins aktive Profil
    • Revert-Button — stellt die gespeicherten Profilwerte wieder her und verwirft die Aenderungen

    Mit ungespeicherten Aenderungen zu starten funktioniert problemlos — der Start nutzt den Live-Arbeitszustand. Das Profil bleibt nur Dirty, bis du committest oder revertest.

    Cross-Reference Wine-Runner-Seite

    Die Wine-Runner-Seite zeigt pro Runner Nutzungs-Badges, damit du auf einen Blick siehst, welche Profile von einem Runner abhaengen, bevor du ihn loeschst: Used by N profile(s)- und Fallback-Badges. Hover ueber das "Used by"-Badge zeigt die Profilnamen.
